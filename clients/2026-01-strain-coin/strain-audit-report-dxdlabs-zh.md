### 6. 发现详情

#### DXD-01 – 中心化无限铸造风险 (High)

**状态:** 未解决

**描述:**  
合约包含带有 `onlyOwner` 修饰符的 `mint(address to, uint256 amount)` 函数。这允许合约所有者无限制地创建新代币。

**影响:**  
所有者可以铸造数十亿个代币并在市场上抛售，导致 STR 代币价值崩盘，给投资者造成严重损失。

**可能性:**  
如果未放弃所有权 (Renounced) 或未将所有权转移到治理合约 (Timelock/Multi-sig)，可能性很高。

**技术详情:**  
```solidity
function mint(address to, uint256 amount) public onlyOwner {
    _mint(to, amount);
}
```

**建议:**  
1. 如果不需要进一步铸造，请放弃所有权 (Renounce Ownership)。
2. 或者将所有权转移到具有时间锁 (Timelock) 机制的多签 (Multi-sig) 合约。
3. 在代码中为总供应量设置硬顶 (Hard cap)。

---

#### DXD-02 – 中心化交易暂停风险 (Medium)

**状态:** 未解决

**描述:**  
合约继承了 `ERC20Pausable`，允许 Owner 调用 `pause()` 函数。在暂停状态下，所有代币转账都将被阻止。

**影响:**  
Owner 可以阻止用户出售其代币，造成类似于“蜜罐”的情况。

**建议:**  
仅在真正的紧急情况下使用此功能，并应建立社区治理机制或设置最大暂停时长限制。

---

#### DXD-03 – 精度 (Decimals) 设置为 0 (Informational)

**状态:** 未解决

**描述:**  
`decimals()` 函数返回 0。

**技术详情:**
```solidity
function decimals() public view virtual override returns (uint8) {
    return 0;
}
```

**影响:**  
大多数 ERC20 代币使用 18 位小数。如果处理不当，使用 0 可能会在钱包或交易所显示时引起混淆。然而，这是一种设计选择，而非安全漏洞。
